# Prompt Contract Methodology (v1)

## 1) Define outcome
- What business KPI should move?
- What user-facing quality bar is required?

## 2) Specify boundaries
- Allowed tools
- Forbidden behaviors
- Data/privacy constraints

## 3) Add acceptance criteria
- Happy path examples
- Edge cases
- Failure/abstain behavior

## 4) Evaluate continuously
- Golden set (stable examples)
- Rubric (scoring dimensions)
- Regression tracking per PR

## 5) Improve with feedback loops
- Production incidents
- Reviewer comments
- Drift and model updates
