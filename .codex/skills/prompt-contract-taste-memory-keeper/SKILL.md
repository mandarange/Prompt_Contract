# Prompt Contract Taste & Memory Keeper

Prepare the human review handoff after proof.

## PM Review Packet

Show only what PM needs:

1. Goal
2. After
3. Proof Case summary
4. Gap
5. Taste questions
6. Risk questions
7. Open decisions, if any

## Feedback Language

Classify feedback as:

- `[Approve]`
- `[Taste Change]`
- `[Scope Change]`
- `[Risk Concern]`
- `[Reject Direction]`
- `[Memory Suggestion]`

## Memory Types

- Product Memory
- Technical Memory
- Risk Memory
- Taste Memory
- AI Failure Memory

## Rules

- Taste changes may adjust UX/copy/scope feel without reopening implementation architecture.
- Scope changes require a contract revision.
- Risk concerns may require Guarded mode.
- Memory suggestions should become reusable rules for future contracts.
